# Структуры в C

**Структуры в C** - это объединенные данные, у которых есть некоторая логическая
взаимосвязь. В отличие от массивов, структуры могут содержать данные разных типов.

### Обычное объявление структуры

```c
struct Passenger {
    int flight_number;
    char name[50];
    char seat[3];
    double baggage_weight;
};
```

Здесь мы создали тег `Passenger`. Чтобы объявить переменную такого 
типа, нужно всегда писать ключевое слово `struct`.

```c
struct Passenger element;
struct Passenger array[10];
struct Passenger *pointer;
Passenger element2;     // ОШИБКА! компилятор не знает, что такое Passenger
```

Так же можно объявлять переменные сразу после объявления структуры:

```c
struct Point {
    double x;
    double y;
} point;
```

### Объявление с помощью typedef

`typedef` позволяет создать псевдоним (другое имя) 
для типа. Применительно к структурам это выглядит так:

```c
typedef struct Point point;

point p;
```

```c
typedef struct {
    double x;
    double y;
} Point;
```

Здесь мы объявили анонимную структуру (без тега) и сразу дали ей имя
`Passenger` через `typedef`. Теперь можно писать просто:

```c
Passenger element;
```

Часто можно встретить комбинированный вариант:

```c
typedef struct Point {
    double x;
    double y;
} Point;
```

Для которого уже будут работать оба обращения:

```c
struct Point element1;
Point element2;
```

### Операции над структурами

А какие же операции можно проделывать со структурами? Ответ на этот вопрос лучше
перечислить по пунктам:
1. присваивание полю структуры значение того же типа
2. можно получить адрес структуры. Не забываем операцию взятия адреса (&)
3. можно обращаться к любому полю структуры
4. для того, что бы определить размер структуры, можно использовать операцию
   `sizeof()`

### Инициализация структуры

Инициализация структуры в языке C происходит так же, как и инициализация
массива. Самый простой способ — перечислить значения полей в фигурных 
скобках в том же порядке, в котором они объявлены в структуре.

```c
struct Point {
    double x;
    double y;
};

struct Point p1 = {10, 20};  // p1.x = 10, p1.y = 20
```

Если какое-либо поле у вас будет не заполненным, то оно автоматом 
заполнится `0` - для целочисленных типов; `NULL` - для указателей;
`\0` (ноль-терминатор) - для строковых типов.

Также доступна именная инициализация:

```c
struct Point p3 = {.y = 20, .x = 10};  // порядок не важен
```

Обращаться в языке C к элементам структуры можно двумя способами:

1. через операцию-точку
   ```c
   struct Point {
       double x;
       double y;
   };
   
   int main() {
   struct Point p = { 12.32, 43.21 };
   printf ("%lf", p.x); //вывод значения элемента liga
   a.y = 2; //присваиваем элементу новое значение
   return 0;
   }
   ```
2. через операцию-стрелку `->`
   ```c
   struct Point *ptr = &p;
   printf ("%lf", ptr->x);
   printf ("%lf", (*ptr).x);
   ```

Различия заключаются в том, что операцию-точку мы используем,
когда дело имеем с переменной, а операцию-стрелку - когда дело
имеет с указателем на переменную.

### Передача структур в функцию

Раз структура является типом данных, только созданным нами, то
с ней можно делать абсолютно такие же взаимодействия, как и с
встроенными типами.

1. структуру можно передавать целиком:
   ```c
   void f(struct Point p){
       printf("%lf %lf", p.x, p.y);
   }
   ```
2. можно передать указатель на структуру:
   ```c
   void d(struct Point *p){
       p->x = 12;
   }
   ```
3. можно передать элементы структуры:
   ```c
   void e(double x, double y){
       return x + y;
   }
   
   struct Point point = { 12.67, 69 };
   e(a.x, a.y);
   ```

# Работа с файлами в языке C

Для программиста открытый файл представляется как
последовательность считываемых или записываемых данных.
При открытии файла с ним связывается **поток ввода-вывода**.

Структура `FILE` содержит необходимую информацию о файле.

## Функции работы с файлами

### fopen

```c
FILE *fopen(const char *name, const char *mode);
```

**Параметры**
- `name` - имя открываемого файла
- `mode` - указатель на строку символов, определяющих способ
 доступа к файлу:

| параметр открытия | значение параметра                                                                              |
|-------------------|-------------------------------------------------------------------------------------------------|
| "r"               | открыть для чтения                                                                              |
| "w"               | открыть пустой файл для записи<br>**если файл существует его содержимое удаляется**             |
| "a"               | открыть файл для записи в конец<br>**файл создается если не существует**                        |
| "r+"              | открыть файл для чтения и записи<br>**файл должен существовать**                                |
| "w+"              | открыть пустой файл для чтения и записи<br>**если файл существует, то его содержимое теряется** |
| "a+"              | открыть файл для чтения и дополнения<br>**если файл не существует, то он создаётся**            |


**Возвращаемое значение**
- `FILE*` - указатель на открытый поток. Если обнаружена ошибка, 
  то возвращается значение `NULL`.

### fclose

```c
int fclose(FILE *File);
```

**Параметры**
 - `file_pointer`: Указатель на файловый объект, который идентифицирует поток, который должен быть закрыт.

**Возвращаемое значение**
- `0`: Когда файл успешно закрыт.
- `EOF`: Когда произошла ошибка при закрытии файла.

### fgetc

```c
int fgetc(FILE *file)
```

**Параметры**
- `file` - указатель на поток типа FILE

**Возвращаемое значение**
- код считываемого элемента
- `EOF` при ошибке

### fputc

```c
int fputc(int ch, FILE *file)
```

**Параметры**
- `file` - указатель на поток типа FILE.
- `ch` - символ для записи

**Возвращаемое значение**
- символ, который записывается
- `EOF` при ошибке

# Работа с бинарными файлами

**Потоки** – это потенциальный источник или приемник 
данных – последовательности байт.

По принципу обработки файловые потоки делятся на потоки
прямого и последовательного доступа. 

В файлах **прямого** доступа все хранимые элементы данные 
занимают одинаковый объем памяти, поэтому получить доступ 
к заданному элементу можно переместиться на нужное расстояние
(смещение).

## Функции работы с бинарными файлами

### fseek

Функция для смещения внутри файлов прямого доступа.

```c
int fseek(FILE *stream, long offset, int whence)
```

**Параметры**
- `stream` - поток
- `offset` - на сколько байт смещать,
- `whence` - откуда (с какой позиции) смещать, константа, принимающая три значения

| значение  | определение        |
|-----------|--------------------|
| SEEK_SET  | от начала          |
| SEEK_CUR  | от текущей позиции |
| SEEK_END  | от конча файла     |

**Возвращаемое значение**
- В случае успеха он возвращает ноль, иначе возвращает ненулевое значение.

### rewind

Функция, используемая для сброса указателя на данный файл в начало файла.

```c
void rewind(FILE *stream)
```

**Параметры**
- `stream` - поток

### ftell

Возвращает текущее значение положения указателя в файле для указанного 
потока. Это значение представляет собой количество байт, на которое указатель 
отстоит от начала файла.

```c
long ftell(FILE *stream)
```

**Параметры**
- `stream` - поток

**Возвращаемое значение**
- количество байт, на которое указатель отстоит от начала файла.
- `-1` - в случае ошибки
- Если в данном потоке невозможен поиск по произвольному адресу
(в случае, например, консоли), возвращаемое значение не определяется.

### fflush

При работе с файловыми потоками обязательно надо помнить, что вывод в
файл осуществляется не напрямую на диск, а через буфер: сначала изменения,
которые должны быть внесены на диск, накапливаются в буфере, а затем, когда
он заполнен, производится операция записи на диск.

```c
int fflush(FILE *stream);
```

**Параметры**
- `stream` - поток

**Возвращаемое значение**
- `0` — при успешном выполнении 
- `EOF` — в случае ошибки

### fread

```c
size_t fread(void *ptr, size_t size, size_t count, FILE *stream);
```

**Параметры**
- `ptr` - указатель на область памяти, которая получает данные из файла
- `size` - размер в байтах одного элемента, который нужно прочитать
- `count` - максимальное количество элементов, которое следует прочитать
- `stream` - поток

**Возвращаемое значение**
- возвращает количество целиком прочитанных элементов
- значение может быть меньше, чем объем, если был достигнут конец файла или произошла ошибка

### fwrite

```c
size_t fread(void *ptr, size_t size, size_t count, FILE *stream);
```

**Параметры**
- `ptr` - указатель на информацию, записываемую в файл
- `size` - размер в байтах одного элемента
- `count` - максимальное количество элементов
- `stream` - поток

**Возвращаемое значение**
- возвращает число записанных элементов
- данное значение равно объем, если не возникла ошибка


### fileno

Некоторые функции языка СИ работают не с переменной потока,
а с дескриптором файла. Дескриптор файла – системный номер файла,
который ему присваивается ОС в таблице открытых файлов. Получить
этот номер можно используя функцию

```c
int fileno(FILE *stream)
```

> **⚠️ ВАЖНО**
>
> Поскольку файлы не интерпретируются, то создать и заполнить
> информацией в блокноте, как это было с текстовыми файлами,
> невозможно. Поэтому каждая программа должна содержать
> функцию создания файла.

## Ответы на вопросы

1) Как найти размер файла

Способ 1: Использование fseek и ftell (Стандартная библиотека)
```c
    fseek(file, 0, SEEK_END); // Перемещаем указатель в конец файла
    long size = ftell(file);   // Получаем позицию (размер файла)
    fclose(file);
```
Способ 2: Использование fstat (POSIX системы - Linux, macOS)
```c
#include <stdio.h>
#include <sys/stat.h>

int main() {
    const char *filename = "example.txt";
    struct stat st;
    
    if (stat(filename, &st) == 0) {
        printf("Размер файла '%s': %lld байт\n", filename, (long long)st.st_size);
    } else {
        printf("Ошибка при получении информации о файле\n");
    }
    
    return 0;
}
```
Способ 3: Использование fligth (Windows)
```c
#include <io.h>
#include <stdio.h>

int main() {
    FILE *file = fopen("example.txt", "rb");
    if (file) {
        int handle = fileno(file); // Получаем дескриптор файла
        long size = filelength(handle); // Получаем размер
        printf("Размер: %ld байт\n", size);
        fclose(file);
    }
    return 0;
}
```

2) Что такое typedef. Можно ли обозначать не только структуры?

typedef — это ключевое слово в C, которое позволяет создавать псевдонимы (синонимы) для существующих типов данных.

Что можно обозначать через typedef?
ДА, можно обозначать не только структуры! typedef работает с любыми типами данных:

```c
typedef int целое;
typedef float дробное;
typedef char символ;

целое a = 10;           // int
дробное b = 3.14;       // float
символ c = 'A';         // char
```

10) У всех ли файлов есть дескриптор?

Файловый дескриптор существует только у открытых файлов в рамках конкретного процесса.

1. Когда файл имеет дескриптор:
 - Только после того, как процесс открыл файл (open(), fopen(), creat() и т.д.)
 - Только для данного процесса (другие процессы могут иметь свои дескрипторы на тот же файл)
 - Только пока файл остается открытым

2. Когда файл НЕ имеет дескриптора:
 - Файл на диске, который никто не открыл
 - Файл после закрытия (close(), fclose())
 - Файлы в файловой системе, к которым нет доступа у процессов

3. Что такое файловый дескриптор?
 - Это целое число (int), которое процесс использует для обращения к открытому файлу
